# Phase 0 Baseline

## Phase 0 Scope

Phase 0 establishes a deployable MVP baseline for the AI CV Fit App without adding new product features. The app keeps its current upload, scoring, result, and DOCX report flow while the repository is prepared for Phase 1 development.

## Current Architecture

- Backend: FastAPI API, SQLAlchemy models, Celery worker, deterministic parsing/scoring services, local/S3-compatible storage abstraction.
- Frontend: simple Jinja template and vanilla JavaScript UI.
- Database: PostgreSQL with pgvector support.
- Queue: Redis used as Celery broker and result backend.
- Reports: DOCX reports generated by the worker and stored through the storage service.
- Local deployment: Docker Compose runs API, worker, PostgreSQL, and Redis.

## Verified E2E Flow

The verified local Docker flow is:

1. Upload a temporary DOCX CV.
2. Create a score job with pasted JD text.
3. Celery worker processes the job from Redis.
4. Worker parses CV/JD, scores the match, stores result JSON, and generates DOCX report.
5. API returns job result JSON.
6. API returns safe report metadata without `local_path`.
7. API downloads a non-empty DOCX report.

## Included In Phase 0

- FastAPI API and current UI.
- CV upload validation for PDF/DOCX.
- Async scoring job with Celery and Redis.
- PostgreSQL persistence for CV metadata, JD text, job state, result JSON, and report location.
- Local/S3-compatible storage abstraction.
- S3 endpoint URL support for S3-compatible providers.
- Safe UUID handling.
- Worker failure handling.
- Render deployment documentation.
- Local Docker smoke test script.
- Repository hygiene for generated uploads/reports/cache files.
